Strategy Pattern é um padrão de design comportamental que permite o encapsulamento de algoritmos alternativos para uma tarefa específica.

Oferecendo uma maneira flexível para escrever diversos algoritmos diferentes, passar para classes clientes que precisam deles.

Esses clientes desconhecem qual é o algoritmo "real" que está sendo executado, e apenas o executam, fazendo com que o código da classe cliente fique desacoplado das implementações concretas de algoritmos, possibilitando com que consiga trabalhar com N diferentes algoritmos sem precisar alterar o seu código.

Exemplificação

No exemplo abaixo nós criamos a classe Commute para encapsular as possíveis estratégias para trabalhar. Então, definimos três estratégias nomeadas Bus, Personal e Taxi.
